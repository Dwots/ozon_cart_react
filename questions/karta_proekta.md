# Карта фронтенда проекта — что есть в проекте и где искать

> Раз теории по React не давали, то «то, что ты знаешь» = то, что реально написано в проекте.
> Здесь: какие файлы, что в них, и на каких строках искать. Открывай и тыкай пальцем.

## 📁 Структура фронтенда (папка `src/`)

```
src/
├── index.js                          ← точка входа (старт приложения)
├── App.js                            ← главный компонент
├── components/
│   ├── Header.jsx                    ← шапка
│   ├── CartDrawer.jsx                ← панель корзины
│   └── AddToCartButton.jsx           ← кнопка "в корзину"
├── pages/
│   └── ProductsPage.jsx              ← список товаров
├── hooks/
│   ├── useCart.js                    ← логика корзины (хук)
│   └── useProducts.js                ← загрузка товаров (хук)
└── data/
    └── products.js                   ← резервный список товаров
```

---

## 1. `src/index.js` — точка входа

С этого файла стартует всё приложение.

- **строки 6-11** — `createRoot(...).render(...)` монтирует `<App>` в `#root`, оборачивает в `<BrowserRouter>` (включает роутинг).

## 2. `src/App.js` — главный компонент

Собирает все кубики вместе и держит общее состояние.

- **строка 11** — `useState(false)` — состояние «открыта ли корзина» (`isCartOpen`).
- **строки 12-22** — берёт всё из хука `useCart()` (корзина, функции, статусы).
- **строки 24-30** — `useEffect` — регистрирует Service Worker (только в production).
- **строки 34-39** — рисует `<Header>`, передаёт ему props.
- **строки 41-45** — `<Routes>` / `<Route>` / `<Navigate>` — роутинг, редирект `/` → `/cart`.
- **строки 47-57** — рисует `<CartDrawer>`, передаёт корзину и функции.

## 3. `src/components/Header.jsx` — шапка

Логотип, статусы (воркер/сеть), кнопка «Корзина» с количеством.

- **строки 3-8** — объект с подписями статусов SharedWorker.
- **строка 10** — приём props через деструктуризацию.
- **строка 16** — `<Link to="/cart">` — ссылка-логотип (без перезагрузки).
- **строки 21-27** — бейджи статусов (воркер + онлайн/офлайн) с условными классами.
- **строка 32** — `{totalCount > 0 && <span>...}` — бейдж с числом товаров (показывается только если >0).

## 4. `src/components/CartDrawer.jsx` — панель корзины

Выезжающее сбоку модальное окно со списком товаров.

- **строки 1-11** — приём кучи props (cart, функции, флаги).
- **строки 14-17** — overlay (затемнение), клик по нему = `onClose`.
- **строка 19** — `<aside>` с условным классом `cart-drawer-open` + `aria-hidden`.
- **строка 27** — `{cart.length === 0 ? ... : ...}` — условный рендер: пусто или список.
- **строки 32-45** — `cart.map(...)` — список товаров, `key={item.id}`.
- **строки 40-42** — кнопки `-` / `+` / «Удалить» (стрелочные функции с аргументом).
- **строки 48-52** — итоги: всего товаров, сумма, кнопка «Очистить».

## 5. `src/components/AddToCartButton.jsx` — кнопка «в корзину»

Самый простой компонент — хороший пример «что такое компонент».

- **строка 2** — props с дефолтным значением `disabled = false`.
- **строки 5-7** — кнопка: `onClick={() => onAdd(product)}`, текст зависит от `disabled`.

## 6. `src/pages/ProductsPage.jsx` — список товаров

Страница каталога.

- **строка 6** — берёт товары из хука `useProducts()`.
- **строки 13-19** — условный рендер статусов: `loading` / `fallback`.
- **строки 22-48** — `products.map(...)` — карточки товаров, `key={product.id}`.
- **строка 23** — `inStock = product.stock > 0` — флаг наличия.
- **строки 27-39** — картинка, название, рейтинг (`toFixed(1)`), цена, наличие.
- **строки 41-45** — `<AddToCartButton>` с `disabled={!inStock}`.

## 7. `src/hooks/useCart.js` — хук корзины (самый важный)

Вся логика корзины. Здесь живут почти все хуки.

- **строки 5-16** — функции `getSavedCart` / `saveCart` (работа с localStorage).
- **строка 19** — `useState(getSavedCart)` — ленивая инициализация корзины.
- **строки 20-25** — `useState` для `isOnline` и `workerStatus`.
- **строка 26** — `useRef(null)` — `portRef` (тихая память для связи с воркером).
- **строки 28-75** — `useEffect` №1: подключение к SharedWorker.
- **строки 78-100** — `useEffect` №2: подписка на события `online`/`offline` + функция очистки.
- **строки 102-199** — функции `addToCart`, `removeFromCart`, `decreaseItem`, `clearCart`.
- **строки 201-207** — `useMemo` для `totalPrice` и `totalCount`.
- **строки 209-219** — `return { ... }` — что хук отдаёт наружу.

## 8. `src/hooks/useProducts.js` — хук загрузки товаров

- **строки 12-13** — `useState` для `products` и `status`.
- **строки 15-45** — `useEffect`: `fetch` товаров, флаг `cancelled`, fallback на резервный список при ошибке.
- **строка 47** — `return { products, status }`.

## 9. `src/data/products.js` — резервные товары

Просто массив из 8 товаров (id, title, price, link, rating, stock). Используется как запасной список, если `fetch` не сработал.

## 🎯 Что значит «я это знаю» — минимум:

1. **Компоненты** — `App`, `Header`, `CartDrawer`, `AddToCartButton`, `ProductsPage` (и что каждый рисует).
2. **Хуки в проекте** — `useState`, `useEffect`, `useRef`, `useMemo` + свои `useCart`, `useProducts`.
3. **Где что лежит** — по таблице выше.
4. **Поток данных** — `useCart` (в `App`) → props вниз в `Header` и `CartDrawer`.
